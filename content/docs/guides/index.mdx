# Build product flows, not just components

PyColors Guides explain **how to use UI primitives in real products**.

They sit between:

- low-level UI components
- real product behavior

And answer the question:

> “How do I turn components into a real SaaS experience?”

---

## What you learn here

Guides focus on the parts of product development that are rarely documented well:

- how components work together
- how UI behaves under real conditions
- how to handle async states
- how to design predictable interactions
- how to structure product surfaces

This is where UI becomes **product experience**.

---

## Where Guides fit

PyColors is structured in layers:

### UI Components

- small, reusable primitives
- token-driven
- implementation-level building blocks

### Guides

- composition of primitives
- UX and behavior rules
- real-world product scenarios
- production constraints

### Starter Pro

- authentication
- billing
- backend
- delivery
- production SaaS foundation

Each layer solves a different problem.

---

## When to use Guides

Use Guides when:

- primitives feel insufficient on their own
- UX consistency becomes important
- you start handling async data
- you want fewer product mistakes
- your app starts behaving like a real product

This is usually the point where:

> UI stops being screens
> and starts becoming a system

## Mental model

> Components define the building blocks.

This layer is what keeps your UI system:

- consistent
- understandable

---

## Practical rule

- Use **UI Components** to build pieces
- Use **Guides** to assemble them correctly
- Use **Starter Pro** to make the product real (auth, billing, backend)
