## Validate before release

Use this checklist before shipping the PWA layer in Starter Pro.

The goal is to validate:

- manifest quality
- icon quality
- screenshot quality
- service worker behavior
- offline fallback routing
- standalone mode
- mobile UX
- production HTTPS behavior

## Manifest checklist

Validate in Chrome DevTools:

```txt title="chrome-devtools.txt"
Application
→ Manifest
```

Confirm:

- app name is correct
- short name is correct
- `display` is `standalone`
- theme color is visible
- background color is visible
- icons are detected
- screenshots are detected
- no installability warning appears

## Asset checklist

| Asset | Expected |
| --- | --- |
| `icon-192.png` | 192x192 |
| `icon-512.png` | 512x512 |
| `maskable-icon-512.png` | 512x512 with safe padding |
| `apple-icon.png` | 180x180 |
| `dashboard-desktop.png` | 1280x720 |
| `dashboard-mobile.png` | 390x844 |

<Callout type="info" title="Screenshot rule">
  Use real dashboard or workspace screenshots. Avoid using only login screens because they do not communicate the full SaaS value.
</Callout>

## Offline checklist

Validate offline behavior:

```txt title="chrome-devtools.txt"
Application
→ Service Workers
→ Offline
```

Then reload the app.

Expected result:

- `/offline` renders
- page keeps PyColors styling
- app does not crash
- reload action is present
- no sensitive stale data is shown

## Mobile checklist

Validate on real devices when possible:

- iPhone Safari
- Chrome Android
- installed standalone mode
- mobile dashboard
- mobile billing page
- mobile settings page
- mobile offline page

## Common questions

<Accordions type="single">
  <Accordion title="Should Lighthouse be perfect before shipping?">
    No. Use Lighthouse as a signal, not as the only release gate.
    The key release gate is whether installability, offline fallback, icons, screenshots, and production HTTPS are correct.
  </Accordion>

  <Accordion title="Should I cache authenticated routes?">
    Not by default. Authenticated SaaS routes can contain sensitive or stale data. Start with fallback-only offline behavior.
  </Accordion>

  <Accordion title="What should I test on iOS?">
    Validate Add to Home Screen, standalone mode, status bar behavior, viewport fit, and mobile navigation.
  </Accordion>
</Accordions>
