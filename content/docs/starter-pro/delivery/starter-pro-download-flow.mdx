## Webhook fulfillment

The system uses Stripe webhooks as the source of truth.

Typical events include:

- `checkout.session.completed`
- `checkout.session.async_payment_succeeded`

This means delivery is triggered by **confirmed payment events**,
not by frontend redirects.

That distinction matters.

---

## Why webhook-first matters

The success page is only a UX surface.

It does **not** guarantee that fulfillment has completed.

With PRO:

- payment confirmation comes from Stripe
- fulfillment runs on the backend
- delivery remains reliable even if the browser closes early

---

## Download token model

Each purchase generates a **secure access token**.

This token is:

- unique
- time-aware
- single-use
- tied to the order flow

It protects the product from being exposed as a public file URL.

---

## Critical design decision

The token is consumed **after** Blob validation.

This prevents a failed file lookup from burning the token too early.

In practice, the flow is:

1. validate token
2. resolve Blob path
3. fetch Blob
4. confirm stream exists
5. consume token
6. stream file to user

This protects the delivery experience from false failures.

---

## Private file storage

Starter Pro is stored in **private Vercel Blob storage**.

The product file is not meant to be served directly from:

- the repo
- `/public`
- a public asset URL
- a GitHub release

Instead, it lives in private storage and is accessed only through the API.

---

## Versioning strategy

The local build artifact can remain simple:

```text
apps/starter-pro/starter-pro.zip
```

But the Blob destination should be versioned:

```text
starter-pro/releases/starter-pro-v1.0.1.zip
starter-pro/releases/starter-pro-v1.1.0.zip
starter-pro/releases/starter-pro-v2.0.0.zip
```

This gives you:

- immutable releases
- rollback safety
- cleaner debugging
- clear delivery history

## Security model

The product file is protected through:

### Private storage

The Blob is not public.

---

### Token-based access

The download route requires a valid token.

---

### Single-use delivery

The same link cannot be reused indefinitely.

---

### API-controlled streaming

The API fetches the file and streams it to the browser.

This avoids exposing the raw Blob URL as a product link.

### Blob

---

### Download
