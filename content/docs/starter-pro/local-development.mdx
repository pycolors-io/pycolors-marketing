## Local environments should feel real

Starter Pro is designed so local development validates realistic SaaS behavior instead of disconnected demo pages.

The local environment supports:

- authentication
- billing
- protected routes
- Prisma
- Stripe
- PWA behavior
- production-shaped workflows

<Callout title="Core principle">
  Local development should validate product infrastructure, not only UI composition.
</Callout>

## Recommended setup flow

<Steps>
  <Step>
    ### Install dependencies

```bash
pnpm install
```
  </Step>

  <Step>
    ### Configure environment variables

```bash
cp .env.example .env.local
```
  </Step>

  <Step>
    ### Start infrastructure

```bash
pnpm db:up
```
  </Step>

  <Step>
    ### Seed and migrate

```bash
pnpm app:setup
```
  </Step>

  <Step>
    ### Start development

```bash
pnpm dev
```
  </Step>
</Steps>

## Recommended validation checklist

Validate:

- authentication flows
- billing routes
- protected pages
- mobile responsiveness
- PWA installability
- offline fallback behavior
- settings surfaces

<Callout type="info" title="Validation principle">
  Validate real product flows locally before customizing deeply.
</Callout>
