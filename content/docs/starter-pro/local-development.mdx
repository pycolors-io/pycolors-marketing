## Local environments should feel real

Starter Pro is designed so local development validates realistic SaaS behavior instead of disconnected demo pages.

The local environment supports:

- authentication
- billing
- protected routes
- Prisma
- Stripe
- PWA behavior
- production-shaped workflows

## Why this matters

Many SaaS products become difficult to scale because local development environments never reflected real production behavior.

That usually creates:

- inconsistent auth flows
- billing drift
- broken onboarding
- unsafe deployment assumptions
- mobile UX regressions
- fragile environment setup

Starter Pro is structured to reduce that operational drift from the start.

```txt title="local-development-model.txt"
Local setup
  ↓
Environment variables
  ↓
Database
  ↓
Auth
  ↓
Billing
  ↓
Protected routes
  ↓
PWA behavior
  ↓
Production confidence
```

<Callout type="info" title="Production insight">

</Callout>

## Recommended setup flow

<Steps>
  <Step>
    ### Install dependencies

```bash
pnpm install
```
  </Step>

  <Step>
    ### Configure environment variables

```bash
cp .env.example .env.local
```
  </Step>

  <Step>
    ### Start infrastructure

```bash
pnpm db:up
```
  </Step>

  <Step>
    ### Seed and migrate

```bash
pnpm app:setup
```
  </Step>

  <Step>
    ### Start development

```bash
pnpm dev
```
  </Step>
</Steps>

Validate:

- authentication flows
- billing routes
- protected pages
- mobile responsiveness
- PWA installability
- offline fallback behavior
- settings surfaces
- local seed fixtures
