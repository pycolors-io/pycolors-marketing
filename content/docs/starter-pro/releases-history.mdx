## Starter Pro release history

This page provides a product-specific release overview for **PyColors Starter Pro**.

For the full PyColors ecosystem history, use the global [changelog](/changelog).

<Callout title="Release philosophy">
  Starter Pro is maintained as a production-ready SaaS foundation.

  Releases focus on authentication, billing, protected application surfaces, reusable SaaS patterns, developer experience, and long-term maintainability.
</Callout>

## Versioning policy

Starter Pro follows Semantic Versioning.

For complete rules and upgrade expectations, see the shared [Versioning policy](/docs/project/versioning-policy).

## Release principles

Starter Pro releases prioritize:

- production readiness
- long-term maintainability
- upgrade safety
- reusable SaaS architecture
- real-world developer workflows
- stable business foundations
- clear product evolution

Features are added progressively with a strong focus on clarity, composability, and operational realism.

<Callout type="info" title="Upgrade philosophy">
  Starter Pro aims to evolve without forcing unnecessary rewrites.

  New capabilities should integrate progressively while preserving upgrade clarity and developer trust.
</Callout>

## Current direction

Starter Pro is evolving from a premium starter into a **production-ready SaaS foundation**.

The current focus is:

- production-ready authentication
- operational Stripe billing
- protected app surfaces
- reusable SaaS workflows
- cleaner onboarding
- stronger buyer confidence
- production-ready UX
- scalable product architecture

## v1.1.0 — Initial commercial release

Released: June 2026

  columns={3}
  tone="platform"

      icon: CreditCard,

      description:
        'Dashboard, projects, admin, billing, settings, pricing, and authenticated navigation.',
      icon: Rocket,
    },
  ]}
/>

### Added

- Email and password authentication
- Google and GitHub OAuth
- Email verification and password reset flows
- Connected account management
- Stripe Checkout and Billing Portal
- Subscription lifecycle handling
- Plan-aware access foundation
- Prisma models for users, auth, billing, and subscriptions
- Docker-based PostgreSQL local setup
- Seed script for local development
- Production-shaped app surfaces

### Migration notes

No migration required.

This release established the first commercial foundation of Starter Pro.

### Notes

This release is the baseline for developers who want to start from a serious SaaS structure instead of rebuilding auth, billing, and app architecture from scratch.

Minor and patch releases are designed to remain backward compatible unless explicitly stated otherwise.

Future major releases will include dedicated migration documentation when required.

  Starter Pro release notes should make upgrade impact clear before developers touch their codebase.
