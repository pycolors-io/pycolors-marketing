## Validate billing before launch

Billing systems require more than checkout UI.

Starter Pro includes production-shaped billing foundations designed to validate:

- Checkout behavior
- webhook synchronization
- subscription lifecycle
- plan-aware product access
- billing-aware navigation

<Callout title="Core principle">
  Redirects are UX. Webhooks are billing truth.
</Callout>

## Stripe CLI

```bash title="stripe-cli.sh"
stripe listen --forward-to localhost:3000/api/webhooks/stripe
```

Then configure:

```bash title=".env.local"
STRIPE_WEBHOOK_SECRET=whsec_xxxxx
```

## Common questions

<Accordions type="single">
  <Accordion title="Why are webhooks required?">
    Because redirects alone cannot confirm what actually happened inside Stripe.
  </Accordion>

  <Accordion title="Should access checks happen client-side only?">
    No. Sensitive billing-aware access should remain server-verifiable.
  </Accordion>
</Accordions>
