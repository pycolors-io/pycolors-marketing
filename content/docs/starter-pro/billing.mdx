## Real monetization flow

This billing layer is designed for a real SaaS flow.

### Upgrade flow

1. user clicks upgrade
2. checkout session is created
3. payment is completed in Stripe
4. subscription state is confirmed
5. webhook sync updates your database
6. the UI reflects real billing state

### Subscription lifecycle

Handled in a real SaaS way:

- trialing
- active
- past due
- canceled

### Billing portal

Users can manage their billing without you building all of it yourself.

That includes flows such as:

- payment method updates
- invoice access
- subscription cancellation
