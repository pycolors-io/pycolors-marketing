# Shadows create depth without noise

PyColors keeps elevation subtle.

A single shadow scale covers the most important surface types:

- controls
- cards
- menus
- modals

The goal is not visual drama.

It is predictable depth.

---

## Shadow scale

| Token | Utility | Typical use |
|---|---|---|
| `shadow-xs` | `shadow-xs` | buttons, pills, inputs |
| `shadow-sm` | `shadow-sm` | default cards and tiles |
| `shadow-md` | `shadow-md` | menus, popovers, floating surfaces |
| `shadow-lg` | `shadow-lg` | modals and strong floating layers |

Each step increases elevation gradually.

That keeps the system calm and avoids visual overload.

---

## How shadows are used

Shadow tokens are defined centrally and mapped into utilities.

Components do not invent custom shadow values.

That means depth stays consistent across:

- light mode
- dark mode
- different surface types
- future product areas

## Usage guidance

- pair shadow level with the right radius and surface type
- use one shadow per surface
- increase elevation only when hierarchy needs it
- reset to `shadow-none` for flat nested elements
- avoid custom `box-shadow` values inside components

If a new elevation level becomes necessary, introduce it as a token.

---

## Why this matters

Shadows are easy to overuse.

When every surface tries to stand out, nothing feels structured.

A controlled elevation system makes the interface feel more deliberate and easier to scan.
