# Typography creates structure before decoration

Typography is one of the strongest signals of product quality.

It defines:

- readability
- hierarchy
- rhythm
- density
- interface clarity

PyColors keeps typography simple, semantic, and predictable so the system remains readable across dashboards, docs, settings pages, and product flows.

---

## Why typography matters

Typography is easy to underestimate because it feels invisible when it works.

But when the type system is weak, the product immediately feels:

- inconsistent
- harder to scan
- more visually noisy
- less deliberate

A strong typography system gives the UI structure before any decorative styling is added.

---

## Font roles

PyColors uses a system-friendly font setup exposed through CSS variables.

Common roles are:

- `font-sans` → default UI and content font
- `font-brand` → optional brand accent font
- `font-mono` → code, technical values, identifiers

A typical setup looks like this:

```tsx
import { Inter } from "next/font/google";

const inter = Inter({
  subsets: ["latin"],
  variable: "--font-sans",
});
```

The important part is not the specific family.

It is the role the font plays in the system.

---

## Default text density

PyColors keeps defaults simple.

- SaaS UI / dashboards → `text-sm`
- Docs / content / marketing → `text-base`

This gives you:

- denser interfaces where needed
- more comfortable reading where content matters

That split is often enough for most products.

## Labels and metadata

Labels are not headings.

They should stay compact, secondary, and structured.

```tsx
<p className="text-xs font-medium uppercase tracking-wide text-muted-foreground">
  Label / metadata
</p>
```

Use this pattern for:

- subtle section labels
- field context
- secondary indicators
- metadata
- small overlines

Labels help orientation without competing with the main hierarchy.

## Semantic text roles

Text should express meaning before visual preference.

| Role | Utility | Usage |
| --- | --- | --- |
| Primary text | `text-foreground` | Main content |
| Secondary text | `text-muted-foreground` | Descriptions, helper text |
| Brand / action text | `text-primary` | Links, highlighted actions |
| Destructive text | `text-destructive` | Errors and dangerous messaging |

These roles should stay stable across the product.

---

## Font weights

| Weight | Utility | Typical use |
| --- | --- | --- |
| 400 | `font-normal` | Body text |
| 500 | `font-medium` | Labels and compact UI emphasis |
| 600 | `font-semibold` | Section headings |
| 700 | `font-bold` | Strong page titles |

The goal is not to use every weight everywhere.

The goal is to keep contrast deliberate.

---

## Practical rules

### Good defaults

- use a clear heading ladder
- keep body text comfortable to read
- reserve smaller text for metadata and helper copy
- use muted text for secondary information
- keep line lengths reasonable in content-heavy surfaces

### Things to avoid

- too many font sizes in one surface
- uppercase body copy
- random heading sizes between pages
- relying on color alone for hierarchy
- using weight as a substitute for structure

## Why this matters in products

Typography is one of the places where systems drift first.

If the hierarchy is unstable:

- pages feel less intentional
- components become harder to scan
- product surfaces lose rhythm

If typography stays stable, the whole product feels calmer and more coherent.

That is why typography belongs to the design system, not to one-off page styling.
