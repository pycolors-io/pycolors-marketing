## Overview

Starter Free is designed to get you from zero to a usable SaaS surface in minutes.

By the end of this guide, you will have a local product environment running with:

- dashboard
- projects
- admin
- billing
- settings
- auth

<Callout title="What Starter Free is for">
  Starter Free helps you validate product direction quickly with a
  production-shaped UI surface, without needing auth, billing, or database
  setup first.
</Callout>

## Prerequisites

Make sure your environment includes:

- **Node.js ≥ 18**
- **pnpm**
- **Git**

Recommended: use the latest Node.js LTS release.

## Installation steps

### Step 1: Clone the repository

```bash
git clone https://github.com/pycolors-io/pycolors-starter-free.git
cd pycolors-starter-free
```

### Step 2: Install dependencies

```bash
pnpm install
```

This installs the local baseline, including:

- Next.js App Router
- Tailwind CSS v4
- PyColors UI
- TypeScript
- ESLint

### Step 3: Start the development server

```bash
pnpm dev
```

Then open:

http://localhost:3000

You should land on the dashboard.

### Step 4: Validate the starter

Open these routes in your browser:

- /login
- /register
- /dashboard
- /projects
- /admin
- /billing
- /settings

If they all load correctly, the installation is complete.

<Callout type="info" title="Expected behavior">
Starter Free is intentionally mocked. You should be able to explore screens,
layouts, and product flows immediately without backend setup or secret configuration.
</Callout>

## First things to try

### Edit the dashboard title

Open:

```
app/dashboard/page.tsx
```

Change the heading text and save.

Hot reload should update the browser instantly.

### Rename the core entity

Search for **Projects** across the repository.

The starter is intentionally easy to rename to your own domain model.

### Modify mock data

Open:

```
components/projects/project-types.ts
```

Change the mock projects to better match your product direction.

## Common pitfalls

Starter Free does not require:

- database URL
- auth secrets
- Stripe keys

Everything is mocked by design.

<Callout title="Good sign">
If the UI loads correctly and route navigation works, your setup is already valid enough
to begin product exploration and UX iteration.
</Callout>

## Troubleshooting

### Port already in use

```bash
pnpm dev --port 3001
```

### Node version errors

```bash
node -v
```

Upgrade to a current Node.js LTS version, then reinstall dependencies.

### Using npm or yarn by mistake

```bash
rm -rf node_modules package-lock.json
pnpm install
```

### Build issues after pulling updates

```bash
rm -rf node_modules .next
pnpm install
pnpm dev
```
