## Overview

Starter Free is designed to get you from zero to a usable SaaS surface in minutes.

By the end of this guide, you will have a local product environment running with:

- auth screens
- dashboard
- projects
- admin
- billing
- settings

No database. No Stripe keys. No auth secrets.

Starter Free is intentionally mocked so you can evaluate the product surface before wiring backend services.

<Callout title="What Starter Free is for">
  Starter Free helps you validate product direction quickly with a production-shaped UI surface, without needing auth, billing, or database setup first.
</Callout>

## Prerequisites

Make sure your environment includes:

- Node.js 18 or newer
- pnpm
- Git

Recommended: use the latest Node.js LTS release.

<Tabs items={["Check versions", "Install pnpm"]}>
  <Tab value="Check versions">

```bash title="terminal"
node -v
pnpm -v
git --version
```

  </Tab>

  <Tab value="Install pnpm">

```bash title="terminal"
npm install -g pnpm
```

  </Tab>
</Tabs>

<Callout type="info" title="Package manager">
  Starter Free is designed around pnpm. Using npm or yarn can create lockfile and dependency inconsistencies.
</Callout>

## Steps

<Steps>

<Step>
### Clone the repository

Clone the starter and move into the project directory.

```bash title="terminal"
git clone https://github.com/pycolors-io/pycolors-starter-free.git
cd pycolors-starter-free
```

</Step>

<Step>
### Install dependencies

Install the local baseline.

```bash title="terminal"
pnpm install
```

This installs the starter stack:

| Dependency area | Included |
| --- | --- |
| Framework | Next.js App Router |
| Styling | Tailwind CSS v4 |
| UI | PyColors UI |
| Language | TypeScript |
| Quality | ESLint |

</Step>

<Step>
### Start the development server

Run the app locally.

```bash title="terminal"
pnpm dev
```

Then open:

```txt title="browser"
http://localhost:3000
```

You should land on the local Starter Free product surface.

</Step>

<Step>
### Validate the starter

Open the core routes:

```txt title="routes"
/
/login
/register
/dashboard
/projects
/admin
/billing
/settings
```

If the routes load correctly and navigation works, the installation is complete.

<Callout type="info" title="Expected behavior">
  Starter Free is intentionally mocked. You should be able to explore screens, layouts, and product flows immediately without backend setup or secret configuration.
</Callout>
</Step>

</Steps>

## Common pitfalls

Starter Free does not require:

- database URL
- auth secrets
- Stripe keys
- email provider credentials
- backend API configuration

Everything is mocked by design.

## Troubleshooting

<Accordions type="single">
  <Accordion title="Port already in use">
    Run the development server on another port.

```bash title="terminal"
pnpm dev --port 3001
```
  </Accordion>

  <Accordion title="Node version errors">
    Check your Node.js version.

```bash title="terminal"
node -v
```

    Upgrade to a current Node.js LTS version, then reinstall dependencies.
  </Accordion>

  <Accordion title="Using npm or yarn by mistake">
    Remove npm lockfiles and reinstall with pnpm.

```bash title="terminal"
rm -rf node_modules package-lock.json yarn.lock
pnpm install
```
  </Accordion>

  <Accordion title="Build issues after pulling updates">
    Clear local generated state and reinstall dependencies.

```bash title="terminal"
rm -rf node_modules .next
pnpm install
pnpm dev
```
  </Accordion>
</Accordions>
